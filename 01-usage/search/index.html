搜索功能

v2.0.0.0+

简易搜索规则：
  • 忽略空格和大小写，模糊搜索，只匹配标题
  • ##双井号开头，搜索除第一个#号以外的内容
  • #12单个井号加数字，跳转到对应序号的服务器
简易搜索示例：

a B C 1 搜索标题包含abc1字符的服务器
##123 搜索标题包含#123字符的服务器
#123 跳转到序号为123的服务器
#，#0 因为#后面的序号不存在，所以跳转到最后一个服务器

高级搜索格式：

#<标签名> <操作符> <参数1> <参数2> ...

文本型标签名说明
name服务器名字
summary服务器摘要(协议@地址)
title服务器标题(序号+名字+摘要)
mark标记
remark备注
tag1-3标签1-3
core(v2.0.0.3+)自定义core名
selected(v2.0.0.3+)是否已选中，内容是字符串 true 或者 false
文本型操作符说明
is完全相同
not完全不相同
has包含
hasnot不包含
like模糊匹配
unlike模糊匹配失败
starts(v2.0.0.5+)匹配开头
ends(v2.0.0.5+)匹配结尾
示例：

#mark is a B C 搜索“标记”是“abc”的服务器。文本类型的参数会合并起来。
#mark abc 搜索“mark”中出现“abc”三个字母的服务器。文本类型默认操作符是“like”。
#mk has abc 因为“mk”可以匹配“mark”和“remark”两个标签名，他们任意一个包含“abc”都会搜到。
#name is "" (v2.0.0.6+)搜索“名称”为空白的服务器。
#name has " " (v2.0.0.6+)搜索“名称”中包含“空格”的服务器。
#name has """ (v2.0.0.6+)搜索“名称”中包含“引号”的服务器。

数字型标签名说明
latency测速延迟(ms)，没测速的是-1
upload上传量(MiB)
download下载量(MiB)
port(v2.0.0.3+)inbound端口号
modify(v2.0.0.3+)修改日期，格式：240230，年份可以省略
数字型操作符说明参数个数
!不等1
=相等1
<小于等于1
>大于等于1
~数值范围2
示例：

#latency > 100 搜索测速结果大于等于100ms的服务器。数字类型的操作符不可以省略。
#latency ! 200 300 搜索测速结果不等于200ms的服务器。后面多余的参数300被忽略掉。
#latency ~ 1 1000 搜索测速结果在[1ms, 1000ms]之间的服务器。
#latency ~ 123 参数个数不足，搜索失败。

快捷键

Ctrl + F 进入搜索框
Esc 离开搜索框

##### 简易搜索规则：
 * 忽略空格和大小写，模糊搜索，只匹配标题
 * `##`双井号开头，搜索除第一个`#`号以外的内容  
 * `#12`单个井号加数字，跳转到对应序号的服务器

##### 简易搜索示例：  
`a  B  C 1` 搜索标题包含`abc1`字符的服务器  
`##123` 搜索标题包含`#123`字符的服务器  
`#123` 跳转到序号为123的服务器  
`#`，`#0` 因为`#`后面的序号不存在，所以跳转到最后一个服务器  

##### 高级搜索格式：
`#<标签名> <操作符> <参数1> <参数2>`

| 文本型标签名 | 说明 |
| -------- | -------- |
| name | 服务器名字 |
| summary | 服务器摘要(协议@地址) |
| title | 服务器标题(序号+名字+摘要) |
| mark | 标记 |
| remark | 备注 |
| tag1-3 | 标签1-3 |
| core | (v2.0.0.3+)自定义core名 |
| selected | (v2.0.0.3+)是否已选中，内容是字符串 true 或者 false |

| 文本型操作符 | 说明 |
| -------- | -------- |
| is | 完全相同 |
| not | 完全不相同 |
| has | 包含 |
| hasnot | 不包含 |
| like | 模糊匹配 |
| unlike | 模糊匹配失败 |

###### 示例：
`#mark is a B C` 搜索“标记”是“abc”的服务器。文本类型的参数会合并起来。  
`#mark abc` 搜索“mark”中出现“abc”三个字母的服务器。文本类型默认操作符是“like”。  
`#mk has abc` 因为“mk”可以匹配“mark”和“remark”两个标签名，他们任意一个包含“abc”都会搜到。  

| 数字型标签名 | 说明 |
| -------- | ------- |
| latency | 测速延迟(ms)，没测速的是-1 |
| upload | 上传量(MiB) |
| download | 下载量(MiB) |
| port | (v2.0.0.3+)inbound端口号 |
| modify | (v2.0.0.3+)修改日期，格式：240230，年份可以省略 |

| 数字型操作符 | 说明 | 参数个数 |
| -------- | ------- | -------- |
| ! | 不等 | 1 |
| = | 相等 | 1 |
| < | 小于等于 | 1 |
| > | 大于等于 | 1 |
| ~ | 数值范围 | 2 |

###### 示例：
`#latency > 100` 搜索测速结果大于等于100ms的服务器。数字类型的操作符不可以省略。  
`#latency ! 200 300` 搜索测速结果不等于100ms的服务器。后面多余的参数300被忽略掉。  
`#latency ~ 1 1000` 搜索测速结果在[1ms, 1000ms]之间的服务器。  
`#latency ~ 123` 参数个数不足，搜索失败。  

##### 快捷键
`Ctrl + F` 进入搜索框  
`Esc` 离开搜索框
